Abstract of the Paper

The question put forward in this paper is whether robots can create conformity. ”Conformity is a change in behaviour or belief as a result of real or imagined group pressure”[S. Kassin, S. Fein, and H. R. Markus. Social Psychology. Cengage Learning, 2013]. To test whether group pressure can be exerted by robots, we recreated a classic social psychology experiment originally conducted by Solomon Asch in the 1950s.

Our experiment has three main dimensions. First, we wanted to know whether robots can create conformity, and whether there is a significant difference between the degree to which individuals conform to a group of robots as opposed to a group of humans. Second, we looked at whether the level of conformity differs between an ambiguous situation and a non-ambiguous situation. And third, we asked if the level of conformity is higher in a visual (line) condition or in a verbal (word) condition.

Our results show that robots can induce conformity but to a significantly lesser degree than humans. Furthermore, the level of conformity is significantly higher in an ambiguous (unclear) situation. And, finally, we found that there is a significant difference between the visual and the verbal task. To conclude, robots are capable of changing our behaviour but are still not as influential as humans can be.

Recreate the experiment

This is a list of things you need to recreate the experiment. I give an introduction how to use all of them

  • Projector on a Machine
  • 4x Nao Robots
  • Machine running a deployd server
  • Machine running the controller (can be same machine)
  • Experiment setting data

The experiment setting data

Find data in the github repo: The setting data

We created for every participant three files: part[x].csv, part[x].js and part[x].xml. The part[x].csv file is a list of all tasks and should be printed out by the experimenter. You should use it to write down if the participant did something right or wrong. We believe a paper is always better than a computer. The part[x].js file includes the lines and words shown to the particpants. This will be loaded by the deployd server. Last but not least, the part[x].xml file which will be loaded by the controller.

Run the presentation on the projector by using deployd and part[x].js file

The following is a list of dependencies. Please install in this order

  1. NodeJs - Is a webserver based on javascript
  2. mongoDB - Is a NoSQL database using JSON to store data (Yes, Javascript. Again!)
  3. deployd - Combines NodeJs and mongoDb to create a simple and fast webserver with REST API
  4. jsonreq - Is an simple JSON request library for NodeJs

Lets assume everything worked fine. What's next?

  1. Download our GitHub repository: our Repostitor by using the following git command: git clone git@github.com:BrandiATMuhkuh/aschgenerator.git or by downloading the zip file.
  2. navigate to the cd aschgenerator/aschgen
  3. run the command dpd. This will start the deployd server
  4. Open your browser and add the following line to your address bar: http://localhost:2403/index.html?file=participents/part1
  5. Now you will see the presentation for participant 1 (part1). Change the number of part[x] at the end of the link to load the presentation for an other participant.
